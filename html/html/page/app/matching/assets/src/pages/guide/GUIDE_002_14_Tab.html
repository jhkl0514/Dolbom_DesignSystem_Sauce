Tab
탭(Tab)은 버튼을 눌러 여러 개의 콘텐츠 섹션을 전환할 수 있는 컴포넌트입니다. 콘텐츠 섹션은 동일한 영역 내에서 전환되기 때문에 정보를 탐색하는 맥락을 유지할 수 있고 작은 공간에 많은 양의 콘텐츠를 효과적으로 표현할 수 있습니다.

Anatomy, 컴포넌트 내부요소

  • 1. Tab
  • 2. Indicator(인디게이터)
  • 3. Label

Type, 종류

1. 라인형
주로 화면의 최상단에 위치하며, 화면의 콘텐츠를 모두 변경하는 탭입니다.
최상의 정보 계층 그룹을 제공합니다. 필요 시, 2depth를 제공할 수 있습니다.
2.세그먼트형
콘텐츠 내 또는 화면 중간의 특정 영역만 변경하는 탭입니다.
최상의 정보 계층 그룹을 제공합니다. 필요 시, 2depth를 제공할 수 있습니다.

State, 상태

on
off

Spec, 제작가이드

1.라인형
  • 항목 너비: 텍스트 길이에 따라 유동적으로 적용.
  • 탭 배치:
  • 한 화면 내 모든 탭을 표시할 경우, 각 탭 항목 간의 패딩 값을 유지.
  • Swipe(스와이프) 또는 좌우 스크롤이 가능한 탭에서는 패딩 값을 유지하고, 너비는 텍스트 길이에 따라 조정.
2.세그먼트형
  • 항목 너비: 화면 내에 모든 탭 항목이 동일한 비율로 균등 배치
  • 탭 배치:
  • 한 화면에 모든 탭이 노출되도록 설계하며, 각 항목의 너비는 고정 비율로 적용. (최대 3개)
Tip.사용법
여러개의 탭이 중첩할 경우, 서로 다른 수준의 탭이 명확하게 구분되도록 사용해야합니다.
1depth는 Tab을 사용하고 2depth는 Filter Chips을 사용합니다.
